# BotDiff UI/UX Specification

Version: 1.0

---

# Design Principles

BotDiff should feel:

- Clean
- Competitive
- Fast
- Professional
- Focused on improvement

The UI should never feel cluttered.

Every page should have one primary action.

---

# Empty States

Every empty page should guide the player.

Example

"No analyzed matches yet."

CTA

Analyze your first ranked game.

---

# Loading States

Every network request should display:

Loading spinner

Skeleton cards

Progress indicator

---

# Error States

Every error should:

Explain what happened.

Explain how to fix it.

Never show raw errors.

---

# Accessibility

Minimum contrast ratio.

Keyboard navigation.

Readable typography.

Large clickable buttons.

---

# Success

The player should always know:

Where they are.

What to do next.

How to improve.
